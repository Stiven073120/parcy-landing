## Especificaciones Técnicas Generales

### Formato:
- **Tipo de archivo:** WebP (optimizado para web)
- **Calidad:** Alta resolución pero optimizada
- **Peso máximo:** 500KB por imagen

### Estilo Visual:
- **Diseño:** Moderno, limpio, profesional
- **Tipografía:** Sans-serif moderna (Inter, Poppins, o similar)
- **Paleta de colores:**
  - Cyan: #06b6d4
  - Azul: #3b82f6
  - Púrpura: #8b5cf6
  - Rosa: #ec4899
  - Verde: #10b981
  - Amarillo: #f59e0b
  - Rojo: #ef4444
  - Gris oscuro: #1f2937
  - Gris claro: #f3f4f6

### Elementos de UI a incluir:
- Bordes redondeados (border-radius)
- Sombras sutiles
- Espaciado generoso
- Iconos modernos
- Tipografía legible
- Contraste adecuado

## Alternativa Temporal

Si necesitas lanzar rápidamente, puedes usar **placeholders** temporales:
1. Usar servicios como Figma, Adobe XD o Sketch para crear mockups
2. Usar bibliotecas de componentes como MUI, Ant Design para screenshots
3. Crear wireframes simples con colores planos
4. Usar herramientas de IA para generar mockups (Midjourney, DALL-E, etc.)

---

## Orden de Prioridad

1. **ALTA:** Imagen principal del Hero (es lo primero que ven)
2. **MEDIA:** Dashboard completo (demuestra valor)
3. **MEDIA:** Plano de disponibilidad (muestra el producto)
4. **BAJA:** Secuencia de proceso (puede funcionar con iconos solo)

---

## Recursos Útiles para Generación

- **Mockup generators:** Placeit, Smartmockups
- **UI kits:** Figma Community, Sketch Resources
- **Screenshot tools:** Cleanshot, Shottr
- **Design inspiration:** Dribbble, Behance (buscar "SaaS dashboard")
- **Color tools:** Coolors.co para mantener paleta consistente

---

## Notas Importantes

1. **Consistencia:** Mantén el mismo estilo visual en todas las imágenes
2. **Branding:** Usa siempre la paleta de colores de Parcy
3. **Realismo:** Las imágenes deben parecer screenshots reales, no renders perfectos
4. **Datos ficticios:** Usa nombres de proyectos, lotes y datos genéricos
5. **Idioma:** Todo el texto dentro de las imágenes debe estar en ESPAÑOL

---
